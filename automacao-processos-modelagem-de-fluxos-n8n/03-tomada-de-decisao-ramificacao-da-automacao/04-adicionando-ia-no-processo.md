# Adicionando IA no processo

Depois da definição do fluxo, pode ser avaliado que em algum momento do processo a utilização da IA pode ser vantajosa, e
para isso pode haver a necessidade utilizar agentes de IA.

Com o N8N isto já é possível! Suas integrações já permitem o uso de IA durante fluxos de processo. Sua implementação é
simplificada por uso de [API Keys](#api-keys) que garante o acesso para aquele modelo de IA.

## Geradores de API Key

Caso seu processo precise de uma integração com IA, porém sua empresa/setor não possuem acesso ou até seja apenas uma
[POC](#poc). Para esses casos, o Gemini permite a criação de uma credencial via <https://aistudio.google.com> gratuitamente.

## Governança

É necessário avaliar com governança se é possível utilizar IA nesta atividade para evitar qualquer tipo de danos e
exposição de dados. Manter uma comunicação assertiva é o segredo para garantir melhores resultados.

## Referências

### API KEYS

>Uma chave de API é uma sequência alfanumérica que os desenvolvedores de API usam para controlar o acesso às suas APIs. Uma API é um mecanismo de comunicação que permite a troca de dados entre dois módulos de software

**AWS**. O que é uma chave de API?. Disponível em: <https://aws.amazon.com/pt/what-is/api-key/>. Acesso em: 17 Maio. 2026.

### POC

> Uma prova de conceito (POC) é o processo de reunir evidências que demonstrem a viabilidade de um projeto. Os gerentes de projeto fazem a POC nas fases iniciais do desenvolvimento, antes de investir muito tempo e recursos em um projeto.

**Atlassian**. Seu guia para prova de conceito (POC) no desenvolvimento do produto. Disponível em: <https://www.atlassian.com/br/work-management/project-management/proof-of-concept>. Acesso em: 17 Maio. 2026.
